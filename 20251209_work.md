### 5. Spec Commands (`mem spec`)

**Features:**

## Database Schema

Currently using the schema from initial migration:

```sql
CREATE TABLE specs (
    id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    file_path TEXT NOT NULL,
    status TEXT DEFAULT 'active', -- active, completed, archived
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

CREATE TABLE tasks (
    id INTEGER PRIMARY KEY,
    spec_id INTEGER,
    parent_id INTEGER, -- NULL for tasks, set for subtasks
    title TEXT NOT NULL,
    file_path TEXT NOT NULL,
    status TEXT DEFAULT 'todo', -- todo, in_progress, blocked, completed
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP,
    FOREIGN KEY (spec_id) REFERENCES specs(id) ON DELETE CASCADE,
    FOREIGN KEY (parent_id) REFERENCES tasks(id) ON DELETE CASCADE
);

CREATE TABLE todos (
    id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT,
    status TEXT DEFAULT 'open', -- open, completed
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

CREATE TABLE work_logs (
    id INTEGER PRIMARY KEY,
    file_path TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Architecture Decisions

### Why GitPython?
- Most popular and well-maintained Python git library
- High-level API easier than subprocess
- Handles all required operations: repo checks, branches, status
- Better error handling
- Will support all Phase 2 git features

### Why Typer?
- Modern CLI framework with automatic help generation
- Type hints for arguments and options
- Easy subcommand structure
- Plays well with other libraries

### Why SQLite + Markdown?
- SQLite for fast querying and relationships
- Markdown for human-editable content
- Best of both worlds: structured data + readable files

### Path Handling
- Store relative paths in database (portable)
- Use absolute paths internally (reliable)
- Display stripped paths to users (clean)
- Clear separation: `caller_dir` vs `mem_working_dir`

## Design Patterns

1. **Command Structure**: Each command group gets its own file in `src/commands/`
2. **Database Helper**: `get_db_connection()` with initialization check
3. **Error Handling**: Consistent use of `typer.Exit(code=1)` for failures
4. **User Feedback**: 
   - Success: ✓ with action description
   - Error: ❌ with helpful message
   - Warning: ⚠️ for non-fatal issues
   - Completion: ✨ for final success
5. **Next Steps**: Always guide users on what to do next
